## 1. 通用规则

| ID | 规则 | 理由 | 例外审批 |
|---|---|---|---|
| CS-0001 | 所有外部接口必须有明确错误码和恢复策略。 | 支持可验证的故障处理。 | 需要 |
| CS-0002 | 禁止未初始化变量、未检查返回值和隐式资源泄漏。 | 降低运行时不确定性。 | 不允许 |
| CS-0003 | 新增并发共享数据必须说明锁、原子操作或无锁依据。 | 防止竞态和优先级问题。 | 需要 |

## 2. Linux 嵌入式规则

| ID | 规则 | 适用范围 |
|---|---|---|
| CS-LNX-0001 | 系统调用、pthread、文件、socket、ioctl 返回值必须检查。 | 用户态/驱动接口 |
| CS-LNX-0002 | 与设备节点、sysfs/procfs、netlink 交互必须记录 ABI 假设。 | 平台接口 |
| CS-LNX-0003 | 日志不得泄露敏感数据，且不得影响实时路径。 | 运行时 |

## 3. RTOS/裸机迁移规则

| ID | 规则 | 迁移关注点 |
|---|---|---|
| CS-RTOS-0001 | 动态内存使用必须可证明有界或替换为静态分配。 | 确定性 |
| CS-RTOS-0002 | 任务、ISR、队列和互斥必须记录最坏路径。 | 时序 |
| CS-RTOS-0003 | 硬件寄存器访问必须集中封装并标注副作用。 | 可测试性 |

## 4. 审查清单

- 是否满足命名、接口、错误处理、资源释放和并发规则。
- 是否有不可验证假设或隐藏平台依赖。
- 是否需要新增需求、设计或测试追踪。
